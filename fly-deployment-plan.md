# Fly.io Deployment Plan

## Context

The site currently runs as a fully static Astro 5 build deployed to Netlify. The goal is to migrate to Fly.io to support future SQLite-backed dynamic routes while keeping existing markdown blog posts pre-rendered (fast, no DB cost on every request). The deployment uses Docker + the `@astrojs/node` adapter in standalone mode. Netlify stays live during a testing period before DNS is cut over.

---

## ~~Phase 1 ‚Äî Switch Astro to hybrid SSR~~ ‚úÖ DONE

**Note:** Astro 5 removed `output: 'hybrid'`. `output: 'static'` now supports per-route SSR opt-in natively ‚Äî it's the same behaviour. The adapter was added without changing the output mode.

Added `@astrojs/node` adapter (standalone mode) to `astro.config.mjs`. All blog pages remain pre-rendered; future SQLite routes use `export const prerender = false`. Verified locally via `node dist/server/entry.mjs` ‚Äî all routes returned 200.

---

## ~~Phase 2 ‚Äî Docker files~~ ‚úÖ DONE

Created multi-stage `Dockerfile` (Node 24 builder + lean runtime) and `.dockerignore` (excludes `node_modules`, `dist`, `.env*`, `.DS_Store`, `.astro`). Server starts on port 8080 via `node ./dist/server/entry.mjs`.

---

## ~~Phase 3 ‚Äî Fly.io app setup~~ ‚úÖ DONE

Created app `shealeslein-com` via `fly apps create`, wrote `fly.toml` (iad region, 256MB shared CPU, auto-stop when idle), and created a 1GB encrypted `sqlite_data` volume mounted at `/data` for future SQLite use.

## ~~Phase 5 ‚Äî DNS cutover~~ ‚úÖ DONE

After testing the fly.dev URL for as long as desired:

### 5a. Add the domain to Fly
```bash
fly certs add shealeslein.com
fly certs add www.shealeslein.com
```

### 5b. Get the IP addresses
```bash
fly ips list
```

### 5c. Update DNS at your registrar
- Point `A` record for `shealeslein.com` to the IPv4 address from Fly
- Point `AAAA` record to the IPv6 address
- For `www`: `CNAME www ‚Üí <appname>.fly.dev` (or same A/AAAA)

### 5d. Monitor cert provisioning
```bash
fly certs check shealeslein.com
```
TLS certs auto-provision via Let's Encrypt once DNS propagates.

### 5e. Decommission Netlify
Once `shealeslein.com` resolves to Fly and HTTPS works, disable or delete the Netlify site.
